Example 4: Callbacks! A Lesson in The Event Loop

Open your console, the examples will show there.

We have three functions, that each call the numbers 1, 2, and 3.
I call them in this order: first(), second(third()), third(). Why does "2" print last?



How it works:

First you need to understand the Stack. The stack holds the code that is currently running. Like a stack of pancakes, think of the stack as executing in "Last in, First Out" order. This means you can add or remove items on the stack only from the very end.

Callbacks are not asynchronous by nature, but they can be used to create asynchronous code.
Callbacks are simply functions that have been passed as the argument to another function. No special code!

function first() {
console.log(1)
}

function second(callback) {
setTimeout(() => {
console.log(2);
// Execute the callback function, but it needs to be optional!
if (callback) {
callback();
}
}, 0)
}

function third() {
console.log(3)
}
In first(), everything is straightforward- console.log the value 1. No funny business here.
In second(), we set it's argument as callback. This is what makes it a callback function.
Then the setTimeout() function runs. It's parameters include an arrow function, and the timer which is set to zero. We set the timer to zero because if the timer wasn't set, the anonymous arrow function would get added to the stack immediately, interrupting other functions currently happening. That has unintended and unpredictable effects. The timer can be set to whatever you want!

In the arrow function, 2 will be logged to the console if there is a callback. We have to check for the callback with an if statement or else it does not run.

In third(), we are simply logging 3 to the console.

When you press the button above, the functions are being called in this order:

first();
second(third());
third();
And what is displayed is:

1
3
3
2

Why is two getting called last when it is the second function we listed?
It's because of the callback! In second(), 3 is logged before two because the callback is present. The first 3 logs, and during the timer, the second 3 logs. Then 2 is logged at the end.

Callbacks are a great example of how to turn sychronous code asynchronous!

Example 5: Array Destructuring

How to make New Data from an Existing Array!

How do I make these three strings go from elements in this array...
const date = ['1970', '12', '01']
... to a separated-by-element date?

How it works:

First, we will declare our date array in our JavaScript file: const date = ['1970', '12', '01']

Then, using the index of each element, we will reassign the elements of date to new variables!
const year = date[0]
const month = date[1]
const day = date[2]

Remember our cute little "donut picker upper"? [ ]
Unfortunately, doing this by hand is tedious and takes up a lot of memory, so there's a better way of doing it called Destructuring.
DESTRUCTURED:
const [year, month, day] = date

Then, log the new variables:
console.log(month)
console.log(year)
console.log(day)



And you have yourself a cute lil destructured array.
